분위수 예측의 개념

예측 구간의 하한은 예측 분포의 10번째 분위수(또는 0.1 분위수)에 해당하며, 실제 값이 이 하한보다 낮을 확률은 약 10%, 높을 확률은 약 90%이다.

보다 일반적으로, 미래 시점 t에서 확률 \(p\)에 해당하는 분위수 예측을 \(f_{p,t}\)라고 하면, 관측값 \(y_t\)는 확률 \(p\)로 \(f_{p,t}\)보다 작을 것으로 기대된다.

예를 들어,
- 10번째 분위수는 \(f_{0.10, t}\)이며,
- 50번째 분위수(중앙값)는 \(f_{0.50, t}\)이다.

Quantile Score (Q점수)

Quantile Score는 다음과 같은 수식으로 계산된다:

\[ Q_{p,t} = \begin{cases} 2(1-p)(f_{p,t} - y_t), & \text{if } y_t < f_{p,t} \\ 2p(y_t - f_{p,t}), & \text{if } y_t \geq f_{p,t} \end{cases} \]

이 점수는 핀볼 손실 함수(Pinball Loss Function)라고도 하며, 그래프의 형태가 핀볼 기계에서 공의 궤적과 유사하다는 데서 유래되었다.

Q점수가 낮을수록 예측이 정확하다는 의미이다.
- p = 0.5일 경우, Q점수는 절대 오차(absolute error)와 동일하다.
- p > 0.5이면 실제값이 예측 분위수보다 클 때 더 큰 패널티가 부과된다.
- p < 0.5이면 실제값이 예측 분위수보다 작을 때 더 큰 패널티가 부과된다.

# 예시 값
p <- 0.1
f_pt <- 744.54
y_t <- 741.84

# Q점수 계산
Q_pt <- if (y_t < f_pt) {
  2 * (1 - p) * (f_pt - y_t)
} else {
  2 * p * (y_t - f_pt)
}

print(Q_pt)
## [1] 4.86

이것은 quantile_score() 함수와 accuracy() 함수를 사용하여 쉽게 계산할 수 있다.

Winkler Score

Winkler Score란?

Winkler Score는 예측 구간을 평가하기 위해 사용되는 지표로, Winkler(1972)에 의해 제안되었다.
예측 구간이 100(1−α)%일 때, 구간은 다음과 같이 정의된다:

\[ [\ell_{\alpha,t}, u_{\alpha,t}] \]

이때, Winkler Score는 다음과 같이 계산된다:

\[ W_{\alpha,t} = \begin{cases} (u_{\alpha,t} - \ell_{\alpha,t}) + 2\alpha(\ell_{\alpha,t} - y_t), & \text{if } y_t < \ell_{\alpha,t} \\ (u_{\alpha,t} - \ell_{\alpha,t}), & \text{if } \ell_{\alpha,t} \leq y_t \leq u_{\alpha,t} \\ (u_{\alpha,t} - \ell_{\alpha,t}) + 2\alpha(y_t - u_{\alpha,t}), & \text{if } y_t > u_{\alpha,t} \end{cases} \]

  • 관측값이 예측 구간 안에 있으면, 점수는 단순히 구간의 길이이다.
  • 관측값이 구간 밖에 있을 경우, 벌점이 추가되며 이는 구간에서 벗어난 정도에 비례한다.

분위수를 활용한 Winkler Score 계산

예측 구간은 분위수를 이용하여 구성할 수 있다.

  • 하한: \(\ell_{\alpha,t} = f_{\alpha/2,t}\)
  • 상한: \(u_{\alpha,t} = f_{1-\alpha/2,t}\)

이때, 두 분위수 점수를 합한 뒤 \(\alpha\)로 나누면 Winkler Score가 된다:

\[ W_{\alpha,t} = \frac{Q_{\alpha/2,t} + Q_{1-\alpha/2,t}}{\alpha} \]

예시 계산

# 값 정의
lower <- 744.54
upper <- 773.22
y_t <- 741.84
alpha <- 0.2

# Winkler Score 계산
W <- if (y_t < lower) {
  (upper - lower) + 2 / alpha * (lower - y_t)
} else if (y_t > upper) {
  (upper - lower) + 2 / alpha * (y_t - upper)
} else {
  (upper - lower)
}

W

Continuous Ranked Probability Score

CRPS란?

대부분의 경우, 하나의 분위수나 예측 구간보다는 전체 예측 분포 자체에 관심이 있다. 이럴 때는 모든 분위수 점수(Quantile Score)를 평균내어 CRPS (Continuous Ranked Probability Score)를 계산한다.
Gneiting & Katzfuss (2014)에 의해 소개된 방법이다.

연속순위확률점수(continuous ranked probability score, CRPS)는 다음과 같이 분위수 점수와 구간 점수의 적분 형태로 표현될 수 있다.

\[ \mathrm{CRPS}(F,y) = \int_0^1 \mathrm{QS}_p(y)\,dp = \int_0^1 \mathrm{IS}_\alpha(F,y)\,w(\alpha)\,d\alpha. \]

여기서 \(\mathrm{QS}_p(y)\)는 분위수 점수(quantile score, pinball loss), \(\mathrm{IS}_\alpha(F,y)\)는 구간 점수(interval score, Winkler score), 그리고 \(w(\alpha)\)는 가중함수(weight function)를 의미한다.

CRPS는 예측 분포 전체를 고려한 가중 절대 오차와 유사하며, 예측된 확률 분포의 신뢰성과 정확성을 함께 평가할 수 있는 지표이다.

google_fc |>
  accuracy(google_stock, list(crps = CRPS))
## # A tibble: 3 × 4
##   .model Symbol .type  crps
##   <chr>  <chr>  <chr> <dbl>
## 1 Drift  GOOG   Test   33.5
## 2 Mean   GOOG   Test   76.7
## 3 Naïve  GOOG   Test   26.5

Scale-free comparisons using skill scores

다양한 시계열 간 예측 정확도 비교

포인트 예측(point forecast)에서처럼, 분포 기반 예측(distributional forecast)에서도 여러 예측 방법을 다양한 스케일의 시계열에 대해 비교하는 것이 중요하다.

  • 포인트 예측에서는 스케일 조정 오차(scaled error)를 사용하여 비교하였다.
  • 또 다른 방법은 Skill Score를 활용하는 것이다.

Skill Score란?

Skill Score는 기준(Benchmark) 방법 대비 성능 향상 비율을 측정하는 지표이다.
분포 예측 정확도를 비교할 때는 CRPS(Continuous Ranked Probability Score)를 기준으로 Skill Score를 계산할 수 있다.

예를 들어, naïve 방법을 기준으로 drift 방법의 성능을 비교하면 다음과 같다:

\[ \text{Skill Score} = \frac{CRPS_{Naïve} - CRPS_{Drift}}{CRPS_{Naïve}} \]

  • 값이 양수이면 drift 방법이 더 나은 성능을 보인다는 의미이다.
  • 값이 0에 가까울수록 기준 방법과 성능 차이가 적다.

accuracy() 함수와 함께 crps() 또는 winkler_score() 함수를 사용하면 Skill Score 계산도 간단하게 수행할 수 있다.

google_fc |>
  accuracy(google_stock, list(skill = skill_score(CRPS)))
## # A tibble: 3 × 4
##   .model Symbol .type  skill
##   <chr>  <chr>  <chr>  <dbl>
## 1 Drift  GOOG   Test  -0.266
## 2 Mean   GOOG   Test  -1.90 
## 3 Naïve  GOOG   Test   0

Skill Score 해석

  • 기준(Benchmark) 방법의 Skill Score는 항상 0이다.
    예: naïve 방법 기준 → naïve의 Skill Score는 0
  • 다른 방법의 Skill Score가 음수이면 기준보다 성능이 나쁜 것이고,
    양수이면 기준보다 성능이 좋은 것이다.

예시:
- drift 방법의 Skill Score가 -0.266이면, naïve 대비 26.6% 더 나쁜 성능을 의미한다.

benchmark 설정 주의사항

skill_score() 함수는 기준 예측 값을 자동 계산한다.
- 시계열이 계절성(seasonal)을 가지는 경우, seasonal naïve가 기준으로 사용된다. - 정확한 비교를 위해, accuracy() 함수에 전달되는 데이터는 훈련 데이터와 같은 시작 시점에서 시작해야 한다.

다양한 지표에 적용 가능

skill_score() 함수는 모든 정확도 지표와 함께 사용 가능하다.

예:

  • skill_score(MSE): 평균제곱오차 비교
  • skill_score(CRPS): 예측 분포 정확도 비교
  • skill_score(RMSE): 평균제곱근오차 비교

5.10 Time series cross-validation

시계열 교차검증(time series cross-validation)은 전통적인 training/test sets보다 더 정교한 방법으로, 다음과 같은 특징이 있다.

  • Test set는 하나의 관측값으로 구성된다.
  • 각 test set에 대응하는 training set는 그 이전 시점까지의 관측값들로 구성된다.
  • 미래 정보는 예측에 사용되지 않는다.
  • 초기 관측값은 training set가 너무 작기 때문에 검증에서 제외된다.

롤링 예측 시작점 (Rolling Forecasting Origin)

이 방법은 “롤링 예측 시작점 평가(rolling forecasting origin)”라고도 하며, 예측 기준 시점이 시간에 따라 한 칸씩 이동하면서 여러 번 평가를 수행한다.

예측 정확도는 모든 test set에 대한 평균 오차로 계산된다.

다단계 예측 (Multi-step Forecasting)

시계열 예측에서는 1단계 예측보다 다단계 예측(예: 4-step-ahead) 이 더 중요한 경우가 많다.
이때는 교차검증 과정도 이에 맞게 수정하여, 여러 단계에 걸친 예측 오차를 측정한다.

Example: Forecast horizon accuracy with cross-validation

gafa_stock 데이터의 google_2015 서브셋은 2015년 동안의 거래일 기준으로 NASDAQ에 상장된 Google Inc의 일별 종가를 포함하고 있으며, 이는 그림 5.9에 시각화되어 있다.

아래의 코드는 1단계부터 8단계 앞까지의 drift 예측 성능을 평가한다. 예상대로, 예측 시차(horizon)가 길어질수록 예측 오차가 증가하는 경향이 그래프에 나타난다.

google_2015_tr <- google_2015 |>
  stretch_tsibble(.init = 3, .step = 1)
fc <- google_2015_tr |>
  model(RW(Close ~ drift())) |>
  forecast(h = 8) |>
  group_by(.id) |>
  mutate(h = row_number()) |>
  ungroup() |>
  as_fable(response = "Close", distribution = Close)
fc |>
  accuracy(google_2015, by = c("h", ".model")) |>
  ggplot(aes(x = h, y = RMSE)) +
  geom_point()
